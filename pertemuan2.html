📝 Pengertian Struktur dan Hierarki Basis Data

  • Struktur : Susunan atau pengaturan unsur-unsur yang saling terkait.
  • Hierarki : Susunan item yang diatur berdasarkan urutan tingkatan atau jenjang, dari yang paling bawah sampai yang paling atas.

Pengertian: Unit terkecil dalam data komputer, bisa bernilai 0 atau 1.
Contoh: Huruf 'A' disimpan sebagai 01000001 dalam bentuk bit.

Pengertian: Satuan terkecil dari informasi bermakna.
Contoh: Nama, NIS, Tanggal Lahir adalah field pada data siswa.

Pengertian: Sekumpulan field yang saling terkait (data dari Field).
Contoh:
NISNamaTanggal Lahir
12345678Ani Wijaya2004-01-15

Pengertian: Kumpulan dari record yang sejenis dan saling berkaitan.
Contoh: data_siswa.
Isi File data_siswa:
NISNamaTanggal Lahir
12345678Ani Wijaya2004-01-15
87654321Budi Santoso2003-10-22

Pengertian: Representasi struktur data dalam bentuk baris dan kolom di dalam basis data relasional.
Contoh: Tabel Siswa yang berisi field NIS, Nama, dan Tanggal Lahir.
NISNamaTanggal Lahir
12345678Ani Wijaya2004-01-15
87654321Budi Santoso2003-10-22

Pengertian: Struktur atau rancangan logis dari basis data yang mendefinisikan bagaimana data diorganisasi dan bagaimana relasi antar data.
Contoh: Skema basis data perpustakaan mencakup tabel Siswa, Buku, dan Peminjaman serta relasi antar tabel tersebut.

Pengertian: Diagram yang menggambarkan hubungan antar entitas dalam sistem.

Komponen ERD:
  • Entitas (kotak): Objek nyata seperti Siswa atau Buku.
  • Relasi (belah ketupat): Hubungan antar entitas, seperti meminjam.
  • Atribut (elips): Informasi yang melekat pada entitas, seperti Nama Buku atau Tanggal Pinjam.

🧠 Aktivitas: Hierarki Data

Hierarki:

Bit → Karakter → Field → Record → File → Tabel → Skema

Ilustrasi:

Bit: 0/1
→ Karakter: 'A'
→ Field: Nama = 'Andi'
→ Record: {NIS: 123, Nama: Andi, Alamat: Bandung}
→ File: Kumpulan data semua siswa
→ Tabel: Tabel Siswa
→ Skema: Kumpulan tabel Siswa, Guru, Kelas

✏️ Contoh ERD Sederhana Sistem Peminjaman Buku

📘 Studi Kasus:

Sistem peminjaman buku di perpustakaan.

🧱 Entitas:
  • Anggota: ID_Anggota, Nama, Alamat
  • Buku: ID_Buku, Judul, Pengarang
  • Peminjaman: ID_Peminjaman, Tanggal Pinjam, Tanggal Kembali
🔗 Relasi:

Anggota meminjam Buku melalui entitas Peminjaman.

🔄 Kardinalitas:
  • 1 Anggota bisa meminjam banyak Buku.
  • 1 Buku bisa dipinjam oleh banyak Anggota (di waktu berbeda).
📊 ERD Sederhana (Skema Teks):
[Anggota] ---< (meminjam) >--- [Buku]
    |                         |
    |                         |
    -------- [Peminjaman] -----
📄 Versi Teks ERD:
Anggota (ID_Anggota, Nama, Alamat)
Buku (ID_Buku, Judul, Pengarang)
Peminjaman (ID_Peminjaman, ID_Anggota, ID_Buku, Tanggal_Pinjam, Tanggal_Kembali)
🖼️ Versi Gambar ERD:

📖 Penjelasan :

🧱 Entitas (Entities)

1. Anggota

Merepresentasikan pengguna atau peminjam buku di perpustakaan.

Atribut:

  • ID_Anggota → Kunci utama (Primary Key), unik untuk setiap anggota.
  • Nama → Nama lengkap anggota.
  • Alamat → Alamat tempat tinggal anggota.

2. Buku

Merepresentasikan koleksi buku yang tersedia di perpustakaan.

Atribut:

  • ID_Buku → Kunci utama, unik untuk setiap buku.
  • Judul → Judul buku.
  • Pengarang → Nama pengarang buku.

3. Peminjaman

Merepresentasikan aktivitas peminjaman buku oleh anggota.

Merupakan entitas relasi (sering disebut entitas lemah) yang menjembatani relasi antara Anggota dan Buku.

Atribut:

  • ID_Peminjaman → Kunci utama untuk identifikasi transaksi peminjaman.
  • ID_Anggota → Kunci tamu (Foreign Key) dari entitas Anggota.
  • ID_Buku → Kunci tamu (Foreign Key) dari entitas Buku.
  • Tanggal_Pinjam → Tanggal buku dipinjam.
  • Tanggal_Kembali → Tanggal buku harus dikembalikan.

🔗 Relasi (Relationships)

🔄 meminjam
  • Menghubungkan entitas Anggota dan Buku melalui entitas Peminjaman.
  • Tipe relasi: Many-to-Many (banyak ke banyak), dipecah melalui entitas perantara Peminjaman.
  • Makna:
    • Seorang anggota dapat meminjam banyak buku.
    • Sebuah buku dapat dipinjam oleh banyak anggota, tapi tidak secara bersamaan (umumnya dicatat berdasarkan waktu peminjaman).

📐 Struktur ERD (Bentuk Visual)

  • Entitas ditampilkan sebagai kotak persegi panjang.
  • Relasi ditampilkan sebagai belah ketupat (diamond) dengan nama relasi "meminjam".
  • Atribut ditampilkan dalam elips yang terhubung ke entitas.
  • Foreign key (kunci tamu) berada pada entitas Peminjaman dan menunjuk ke Anggota dan Buku.

💡 Simulasi Sistem Berjalan

1. Input Data Anggota

Seseorang mendaftar menjadi anggota perpustakaan.

📥 Input ke Tabel Anggota:

INSERT INTO Anggota (ID_Anggota, Nama, Alamat)
      VALUES ('A001', 'Dewi Lestari', 'Jl. Mawar No.10');
2. Input Data Buku

Petugas perpustakaan menambahkan data buku ke sistem.

📥 Input ke Tabel Buku:

INSERT INTO Buku (ID_Buku, Judul, Pengarang)
      VALUES ('B001', 'Laskar Pelangi', 'Andrea Hirata');
3. Peminjaman Buku

Anggota A001 meminjam buku B001 pada tanggal 10 April 2025, dengan batas pengembalian 17 April 2025.

📥 Input ke Tabel Peminjaman:

INSERT INTO Peminjaman (ID_Peminjaman, ID_Anggota, ID_Buku, Tanggal_Pinjam, Tanggal_Kembali)
      VALUES ('P001', 'A001', 'B001', '2025-04-10', '2025-04-17');
4. Melihat Data Peminjaman

Petugas ingin melihat semua peminjaman yang sedang aktif.

📤 Query:

SELECT P.ID_Peminjaman, A.Nama, B.Judul, P.Tanggal_Pinjam, P.Tanggal_Kembali
FROM Peminjaman P
JOIN Anggota A ON P.ID_Anggota = A.ID_Anggota
JOIN Buku B ON P.ID_Buku = B.ID_Buku;

🖥️ Output:

ID_Peminjaman Nama Judul Tanggal_Pinjam Tanggal_Kembali
P001 Dewi Lestari Laskar Pelangi 2025-04-10 2025-04-17
5. Pengembalian Buku

Saat buku dikembalikan, data bisa dihapus atau statusnya diubah.

📥 Contoh Update Status:
(jika tabel punya kolom Status):

UPDATE Peminjaman
SET Status = 'Dikembalikan'
WHERE ID_Peminjaman = 'P001';

📝 Rangkuman Pertemuan 2: Struktur dan Hierarki Basis Data

  • Struktur Hierarki Basis Data: Data disusun dalam beberapa level mulai dari:
    • Bit: Unit terkecil data, berupa 0 atau 1.
    • Field: Sekumpulan karakter, mewakili satu atribut (misalnya: nama mahasiswa).
    • Record: Kumpulan field, mewakili satu entitas (misalnya: satu data mahasiswa).
    • File: Kumpulan record yang memiliki format sama (misalnya: file data seluruh mahasiswa).
    • Tabel: Struktur yang menyimpan data dalam bentuk baris (record) dan kolom (field).
    • Basis Data: Kumpulan tabel yang saling berelasi dan membentuk satu sistem informasi.
  • Skema Basis Data: Gambaran logis tentang struktur basis data yang mencakup tabel, atribut, tipe data, dan hubungan antar tabel.
  • Entity Relationship Diagram (ERD): Diagram yang menggambarkan entitas, atribut, dan relasi antar entitas dalam sistem basis data. Digunakan sebagai langkah awal dalam merancang struktur basis data.
  • Relasi Antar Tabel: Dihubungkan menggunakan primary key dan foreign key, misalnya:
    • Tabel Mahasiswa memiliki NIM sebagai primary key.
    • Tabel Peminjaman memiliki NIM sebagai foreign key yang mengacu ke tabel Mahasiswa.
  • Studi Kasus: Sistem peminjaman buku perpustakaan:
    • Tabel Mahasiswa: Menyimpan data mahasiswa.
    • Tabel Buku: Menyimpan data koleksi buku.
    • Tabel Peminjaman: Menyimpan catatan peminjaman yang menghubungkan mahasiswa dengan buku.
    • Implementasi SQL digunakan untuk membuat tabel, relasi, dan menyimpan data.
