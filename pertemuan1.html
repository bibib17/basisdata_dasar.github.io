Pengertian Basis Data

1. Definisi Basis Data

Basis data adalah kumpulan data yang saling berhubungan dan disimpan secara sistematis agar mudah diakses, dikelola, dan diperbarui. Tujuannya adalah untuk menyediakan informasi yang dibutuhkan secara efisien. (ambil contoh gudang yang tersusun)

Fungsi database (basis data) sangat penting dalam berbagai sistem informasi karena menjadi pusat penyimpanan, pengelolaan, dan pengambilan data.

Dengan bantuan DBMS, database dapat melakukan:

  • Insert (menambahkan data baru)
  • Update (memperbarui data)
  • Delete (menghapus data)

Database menjaga agar data tetap valid, konsisten, dan akurat melalui:

  • Relasi antar tabel (foreign key)
  • Normalisasi data

ğŸ”¹ 4. Mengurangi Duplikasi Data

2. DBMS (Database Management System)

Merupakan perangkat lunak yang digunakan untuk mengelola basis data. DBMS itu ibarat otak dari sistem basis data â€” dia yang ngatur semua proses agar data bisa disimpan, diakses, diubah, dan dijaga dengan baik. Contoh DBMS:

  • MySQL
  • PostgreSQL
  • Oracle
  • SQL Server
  • MongoDB (untuk NoSQL)
Catatan: Pemilihan DBMS yang tepat sangat penting untuk menunjang performa dan skala sistem informasi yang dibangun.

ğŸ”§ Peran Utama DBMS

1. Menyimpan dan Mengelola Data

DBMS menyimpan data dalam bentuk terstruktur (biasanya tabel), dan mengelola semua proses CRUD:

  • Create (buat data)
  • Read (baca data)
  • Update (ubah data)
  • Delete (hapus data)
2. Mengontrol Akses ke Data

DBMS memastikan hanya pengguna yang berwenang yang bisa akses atau mengubah data:

  • Hak akses (read/write/admin)
  • Otentikasi pengguna
  • Role-based access (contoh: admin, user, guest)
3. Menjaga Keamanan dan Integritas Data
  • Mencegah data yang tidak valid masuk (misal angka di kolom nama)
  • Menjaga relasi antar data tetap konsisten
  • Mencegah duplikasi dan inkonsistensi data
4. Memfasilitasi Transaksi Data

DBMS mendukung ACID properties dalam transaksi:

  • Atomicity: Semua langkah dalam transaksi harus sukses atau gagal semua
  • Consistency: Data tetap valid sebelum dan sesudah transaksi
  • Isolation: Transaksi berjalan tanpa gangguan dari transaksi lain
  • Durability: Data tetap tersimpan meski sistem crash
5. Backup dan Recovery
  • Backup data secara berkala
  • Restore data jika sistem rusak atau error
6. Mengoptimalkan Kinerja Query

DBMS punya sistem untuk menyusun strategi eksekusi query terbaik (query optimizer), misalnya:


SELECT * FROM order WHERE user_id = 1;

Hasilnya bisa ditampilkan dengan cepat karena DBMS tahu indeks mana yang dipakai.

7. Mendukung Multi-User Access

DBMS mengatur agar banyak pengguna bisa mengakses dan memodifikasi data secara bersamaan tanpa konflik.

8. Abstraksi Data

DBMS menyembunyikan detail teknis penyimpanan data dari pengguna, jadi developer hanya fokus pada struktur logisnya saja, bukan teknis penyimpanannya.

3. Tabel

Dalam basis data relasional, tabel adalah struktur dasar yang digunakan untuk menyimpan data dalam baris (row) dan kolom (column).

ID Nama Email
1 Budi budi@email.com
2 Sinta sinta@email.com

4. Record dan Field

Record (baris): satu unit data lengkap dalam tabel.
Field (kolom): atribut atau karakteristik dari data.

5. Primary Key

Merupakan identitas unik dari setiap record dalam tabel. Tidak boleh null dan harus unik.
Contoh: kolom ID pada tabel pengguna.

6. Foreign Key

Digunakan untuk menghubungkan dua tabel, menunjuk ke primary key dari tabel lain.

7. Relasi Antar Tabel

Basis data relasional memungkinkan relasi antar tabel:

  • One-to-One
  • One-to-Many
  • Many-to-Many

8. SQL (Structured Query Language)

Bahasa standar untuk mengakses dan memanipulasi basis data relasional. 


      SELECT * FROM users WHERE email = 'budi@email.com';

9. Normalisasi

Proses untuk mengorganisasi data dalam database agar menghindari duplikasi dan memastikan integritas data.

10. Keamanan dan Akses

Basis data menyediakan mekanisme kontrol untuk:

  • Hak akses pengguna
  • Enkripsi data
  • Backup dan recovery
